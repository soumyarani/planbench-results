CooperBench Sub-Task (Hunk) Level Interaction Analysis

Decomposes each feature patch into individual hunks (sub-tasks) and analyzes hunk-to-hunk interactions across agents. Python repos only.

Methodology

Patch-level analysis treats each agent's entire patch as one atomic subgoal. This hunk-level analysis decomposes each patch into individual hunks (one per @@ header) and analyzes ALL cross-agent hunk pairs.

For each hunk, we extract using regex:

  • Which file it modifies
  • What symbols it WRITES (function defs, class defs, method defs, imports, variable assignments)
  • What symbols it READS (function calls, name references)
  • The enclosing scope (function/class from context lines)

Each hunk pair is classified into one of 5 categories:

CategoryDescriptionColor
INDEPENDENTNo symbol overlap, no line overlapGreen
SYNERGYShared imports (idempotent) or unidirectional write-readBlue
DIRECT_DESTRUCTIVEWrite-write on real symbols or overlapping lines with conflictOrange
TRANSITIVE_DESTRUCTIVECross-file dependency through importsPurple
NON_SERIALIZABLEBidirectional write-read or overlapping lines + bidirectional symbolsRed

This means the "non-serializable" problem is far more localized than patch-level analysis suggests. A smart merge tool only needs to handle specific conflicting hunks, not rewrite entire patches.
